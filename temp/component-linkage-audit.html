Component linkage audit

Working doc for revising linked property groups across all 14 component editors. Component order matches the editor sidebar (alphabetical by registry label).

Reference pattern (from Button)
  • Link shape props (border-width, radius, padding, similar geometry) across all variant×state cells.
  • Link typography props (font-family, font-size, font-weight, line-height) across whatever axis varies.
  • Don't link colors — surface, border, text, fill. They're meant to vary.
  • Wrap text color + typography in a TypeGroupConfig so they read as one fieldset.
GroupKey naming convention (resolved 2026-05-07)
  • Drop component-level prefixes (card-, tab-) — linkage is already scoped per-component slice; the prefix is redundant.
  • Keep part-level prefixes when multiple parts coexist (e.g. title- vs body- in Card; label- vs value- in Progress Bar).
  • Drop part-level prefixes when only one part exists (e.g. label- in Collapsible Section, tab- in Tab Bar).
Cross-cutting decisions — all resolved (2026-05-07)
  • Inline Edit Actions — keep split: save and cancel keep separate save-*/cancel-* linked groups (cancel allowed to differ from save).
  • Segmented Control radius pair — keep distinct, unlinked: bar-radius and selected-radius stay separate.
  • Tab Bar bar-level tokens — --tabbar-bar-radius, --tabbar-bar-divider, --tabbar-bar-divider-thickness, and --tabbar-bar-padding are all surfaced in the schema as unlinked single tokens.
  • Section Divider — skip: no linkage, editor stays as-is.
Reading the matrix

Properties on the left (rows). Variant/state/part cells across the top (columns). Hover any cell for its CSS variable name.

The four cell states distinguish does this cell have its own editable variable, and is it linked to others:

  • A / B / C … (colored letter) — has its own variable, and it's a member of a linked group. Same letter elsewhere = same group; editing any one cell propagates to every cell with that letter.
  • · (dot) — has its own variable, but is unlinked. Editing it changes only that one cell. Use this when the property is meant to vary per cell (e.g. surface/border/text colors that differ across states by design).
  • → (faded letter color) — has no variable of its own. The value cascades in from another cell in the same row (typically the per-variant cell carrying that property). Cell can't be edited directly, but the value visually matches the rest of the group.
  • — (striped) — property genuinely doesn't apply here (e.g. Dialog's overlay row has no border or title typography at all).
